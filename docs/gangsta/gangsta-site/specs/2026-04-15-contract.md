# Contract: Gangsta Documentation Site

## Objective

Build a dual-purpose documentation site for the Gangsta framework using Docus. The site serves two audiences: (1) human developers who need to learn, reference, and configure Gangsta; (2) AI agents who install Gangsta skills via the `npx skills add` protocol. The site is deployed as a static site on Cloudflare Pages, with build-time synchronization of skill files from the canonical GitHub repository.

## Requirements

### Functional Requirements

**FR-001** Landing Page
The site shall present a landing page at `/` with a hero section, feature highlights (Spec Before Code, 20 Skills, 1% Rule, Multi-Agent Orchestration, 5 Platforms, Institutional Memory), the Heist Pipeline visualization, the Borgata Hierarchy diagram, and CTAs linking to Getting Started and GitHub.

**FR-002** Documentation Section — Getting Started
The site shall provide a "Getting Started" section containing: overview of Gangsta, platform-specific installation guides (5 platforms: Claude Code, OpenCode, Codex, Gemini CLI, Cursor), a quickstart guide for running a first Heist, and an explanation of the 1% Rule.

**FR-003** Documentation Section — Core Concepts
The site shall provide a "Core Concepts" section covering: Borgata Hierarchy, Heist Pipeline overview, Omerta Laws, Intent Routing, The Ledger, and Spec-Driven Development philosophy.

**FR-004** Documentation Section — Heist Pipeline Deep-Dive
The site shall provide a "Heist Pipeline" section with dedicated pages for each of the 6 phases (Reconnaissance, The Grilling, The Sit-Down, Resource Development, The Hit, Laundering), including the phase's purpose, process, required checklist, outputs/artifacts, and gate conditions. Each phase page shall display a "Phase Gate" callout indicating Don approval is required.

**FR-005** Documentation Section — Skills Reference
The site shall provide a "Skills Reference" section auto-generated from the canonical GitHub skill files. The number of skills served shall match the upstream source at build time (not hardcoded). Each skill page shall display: the skill name, invocation command, type classification (hierarchy/heist/dev), purpose description, process/checklist, and outputs. Skills shall be organized into three sub-categories: Hierarchy Roles, Heist Phases, and Dev Skills.

**FR-006** Documentation Section — Advanced
The site shall provide an "Advanced" section covering: custom skill authoring, multi-agent orchestration patterns, checkpoint recovery (resuming interrupted Heists), and contributing guidelines.

**FR-007** Agent Skills Endpoint
The site shall serve Gangsta skills via Docus's native Agent Skills module at `/.well-known/skills/`. Docus's skills module operates at build time: it scans the `skills/` directory, registers Nitro prerender routes, and pre-renders all skill endpoints as static assets during `nuxt generate`. This is fully compatible with static-only deployment (NFR-003) — no runtime server routes are required. The `skills/` directory shall be populated at build time from the GitHub repository `kucherenko/gangsta`, path `skills/` (branch: `main`). The endpoint shall serve both the discovery index (`/.well-known/skills/index.json` — JSON with `Content-Type: application/json`) and individual skill files as raw Markdown (`Content-Type: text/markdown`) including subagent prompt references (e.g., `/.well-known/skills/the-grilling/proposer-prompt.md`).

**FR-008** GitHub Auto-Sync
A build-time script shall pull skill files from `kucherenko/gangsta` (GitHub REST API, `skills/` path, branch: `main`) and: (a) copy raw files to the site's `skills/` directory for Agent Skills serving; (b) generate Docus content pages in `content/skills/` with injected navigation frontmatter. The sync script shall run as a pre-build step before static generation. On GitHub API failure (unavailable, rate-limited, or path change), the build shall fail loudly with a descriptive error message (never silently proceed with stale content). A `GITHUB_TOKEN` environment variable shall be required; the build shall fail if unset.

**FR-009** Push-Triggered Rebuild
A Cloudflare Pages deploy hook shall be configured as a GitHub webhook on the `kucherenko/gangsta` repository, triggered by pushes to `skills/**` on the `main` branch. This ensures the documentation site rebuilds and re-syncs whenever skill files change upstream. The webhook shall use GitHub's `X-Hub-Signature-256` validation against a stored secret to prevent unauthorized rebuild triggers.

**FR-010** Heist Pipeline Visualization
The site shall display a static SVG diagram of the 6-phase Heist Pipeline (Reconnaissance → The Grilling → The Sit-Down → Resource Development → The Hit → Laundering) showing phase names, skill invocations, and gate artifacts. This component shall appear on the landing page and the Pipeline overview page.

**FR-011** Borgata Hierarchy Diagram
The site shall display a static SVG diagram of the Borgata Hierarchy (Don → Consigliere + Underboss → Capo → Soldier/Associate with The Ledger as a side element). This component shall appear on the landing page and the Core Concepts hierarchy page.

**FR-012** Site Search
The site shall include Docus's built-in full-text search, enabled by default, covering all documentation content.

### Non-Functional Requirements

**NFR-001** Dark Mode Forced
The site shall enforce dark color mode as the only mode. No light/dark toggle shall be displayed. The noir aesthetic is the identity, not a preference.

**NFR-002** Gold/Amber Accent Color System
The site shall use a custom gold color palette (11 shades, defined via Tailwind CSS 4 `@theme` in oklch color space) as the primary accent color. Docus's `--ui-*` CSS custom properties shall be remapped to the gold palette. Background surfaces shall use near-black tones with cool undertones. Shades designated for interactive text and body copy must achieve a 4.5:1 contrast ratio against the background per WCAG AA. Decorative/accent-only shades are exempt from contrast requirements.

**NFR-003** Static Deployment
The site shall be fully statically generated via `nuxt generate` and deployed to Cloudflare Pages. No server-side rendering or dynamic server routes shall be required at runtime (Agent Skills routes are pre-rendered during build).

**NFR-004** Node.js Version Pinning
The project shall include `.nvmrc` and `.node-version` files pinning to Node 20. The `package.json` shall declare `"engines": { "node": ">=20" }`. Cloudflare Pages build environment shall be configured for Node 20.

**NFR-005** Dependency Version Pinning
All dependencies (`docus`, `nuxt`, `better-sqlite3`, `@octokit/rest`, `tsx`) shall be pinned to specific versions in `package.json`, not `"latest"`.

**NFR-006** Accessibility
The SVG diagram components shall include `aria-label` and `role="img"` attributes. All color contrast ratios shall meet WCAG AA standards. Navigation shall be keyboard-accessible (Docus provides this by default).

**NFR-007** Performance
The static site shall achieve Lighthouse scores of 95+ on Performance, Accessibility, and SEO; 90+ on Best Practices. No client-side JavaScript frameworks or heavy runtime dependencies shall be added beyond what Docus provides.

**NFR-008** SEO
Each page shall have unique `title` and `description` meta tags. OG images shall be auto-generated by Docus. A sitemap shall be auto-generated. The canonical site URL shall be configured for proper indexing.

## Architectural Decisions

### AD-001: Docus as Framework
**Decision**: Use Docus (Nuxt 4 layer) as the site framework.
**Rationale**: Native Agent Skills support, built-in dark mode, auto-generated sidebar, MDC landing page components, built-in search, Nuxt ecosystem extensibility. No other static site generator has native `/.well-known/skills/` serving.

### AD-002: Dual-Purpose Architecture (Docs + Install Endpoint)
**Decision**: The site serves both human-readable documentation and machine-installable Agent Skills.
**Rationale**: The `npx skills add https://gangsta.dev` command becomes the primary installation method. This eliminates the need for users to clone the GitHub repo and manually configure skill paths. The Docus `skills/` directory is the native mechanism.

### AD-003: GitHub Auto-Sync, Not Manual Content
**Decision**: Skill content is pulled from `kucherenko/gangsta` at build time, not hand-authored.
**Rationale**: Eliminates the three-sources-of-truth problem. One canonical source (GitHub), two derived representations (machine-readable `/skills/`, human-readable `/content/skills/`). Reduces maintenance burden from 40-80 hours of initial writing to an automated pipeline.

### AD-004: Static SVG Components, Not Interactive
**Decision**: Custom components (Pipeline Flow, Hierarchy Tree) use static SVG embedded in Vue SFCs.
**Rationale**: Zero SSR complexity. No `<ClientOnly>` needed. No hydration mismatch risk. Renders identically at build time and runtime. 4-8 hours per component vs 20-40 hours for interactive alternatives.

### AD-005: Cloudflare Pages Deployment
**Decision**: Deploy as static site to Cloudflare Pages via `nuxt generate`.
**Rationale**: Free tier sufficient. Global CDN. Push-triggered rebuild via deploy hooks. No server-side SQLite concerns.

### AD-006: Forced Dark Mode
**Decision**: Color mode is forced to dark. No toggle.
**Rationale**: The noir aesthetic is the brand identity. Allowing light mode would undermine the visual concept. `docus.colorMode: { preference: 'dark' }` with no toggle.

### AD-007: Learning Path Navigation Structure
**Decision**: Documentation organized by learning path: Getting Started → Core Concepts → Heist Pipeline → Skills Reference → Advanced.
**Rationale**: Serves the primary user journey (new user onboarding → deep-dive → reference lookup). Navigation ordering via `.navigation.yml` per-section and `navigation.order` frontmatter per-page.

## Grilling Conclusions

### Key Decisions
- **Dual-purpose site**: Both human docs and Agent Skills install endpoint — the `skills/` directory is a primary deliverable
- **Auto-sync from GitHub**: `kucherenko/gangsta` `skills/` path, via build-time script using GitHub API
- **Static SVG**: Pipeline and hierarchy diagrams are pure markup, no interactivity
- **Cloudflare Pages**: Static deployment with push-triggered rebuild webhook
- **Force dark mode**: Gold/amber on black, no toggle
- **Navigation via .navigation.yml**: Not numbered URL prefixes; `.navigation.yml` per section + `navigation.order` frontmatter per page

### Rejected Alternatives
- **Light mode toggle**: Rejected — noir is the brand, not a preference
- **Interactive D3/Mermaid diagrams**: Rejected — 5x implementation cost, SSR complexity, no proportional value gain
- **Numbered URL prefixes** (e.g., `/1.quickstart/...`): Rejected — Docus v4 uses `.navigation.yml` ordering, numbered prefixes risk numbered URLs
- **Hand-authored skill docs**: Rejected — 40-80 hours commitment, guaranteed content drift vs canonical source
- **Single-page tabbed platform guides**: Rejected — diverging content per platform makes tabs unmaintainable
- **SSR deployment** (`nuxt build` + Node server): Rejected — SQLite incompatibility with serverless runtimes, unnecessary complexity for static content

### Unresolved Objections
- **Docus `--ui-*` token coverage**: The CSS remapping covers primary tokens. Docus component layer may have additional tokens (`--ui-card-*`, `--ui-prose-*`) that need visual QA after first deployment. — Risk: LOW — Mitigation: Visual QA pass post-deployment.
- **Skills sync API rate limits**: 60 req/hr unauthenticated for GitHub API. With 19 skills (some with sub-files), unauthenticated builds could hit limits. — Risk: MEDIUM — Mitigation: Set `GITHUB_TOKEN` env var in Cloudflare build environment (5000 req/hr).

5. **AC-005**: `/.well-known/skills/reconnaissance/SKILL.md` returns the canonical reconnaissance skill content

8. **AC-008**: Dark mode is forced — no color mode toggle visible
9. **AC-009**: Gold/amber accent color renders correctly on primary UI elements (buttons, links, sidebar active items)
10. **AC-010**: SVG diagrams render correctly on both mobile (vertical stack) and desktop (horizontal) viewports
11. **AC-011**: Full-text search works across all content pages
12. **AC-012**: Platform-specific installation guides exist for all 5 platforms
13. **AC-013**: Each Heist Phase page displays the Phase Gate callout
14. **AC-014**: Site deploys to Cloudflare Pages and serves via CDN

## Out of Scope

- **Multi-language/i18n**: Single English language only for initial release
- **AI-powered chat assistant**: Docus supports this but it's not in scope
- **User authentication or dynamic content**: Fully static site
- **Custom domain DNS configuration**: Infrastructure task, not code
- **Logo/brand asset design**: Static assets (SVG logo, favicon, OG image) will use placeholder or basic design; professional brand design is out of scope
- **Content authoring for non-skill pages**: Hand-authored pages (Getting Started, Core Concepts, Pipeline, Advanced) are in scope but drafting is a content task, not a code task
- **Automated visual regression testing**: Manual QA for initial release
